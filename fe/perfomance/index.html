# 函数优化

  • lazy parsing 懒解析 vs eager parsing 解饿解析
  • 为了优化加载速度，浏览器对函数一般采取lazy parsing 懒解析方式（调用时才会解析）。
  • 但是某些情况下需要立即执行该函数，会触发浏览器对函数进行eager parsing 饥饿解析
  • 如何直接进行饥饿解析，看如下代码
export default () => {
  const add = (a,b) => a + b;
  const num1 = 1;
  const num2 = 2;

  add(num1,num2);
}

// 给函数+()告诉解析器，要对函数进行饥饿解析
export default () => {
  const add = ((a,b) => a + b);
  const num1 = 1;
  const num2 = 2;

  add(num1,num2);
}

# 对象优化

TIP

V8中的隐藏类（Hidden Classes）和内联缓存（Inline Caching） (opens new window)

  • 以相同顺序初始化对象成员，避免隐藏类的调整
  • 实例化后避免添加新的属性
  • 尽量使用Array代替array-like对象
class RectArea { // HC0
  constructor(l,w) {
    this.l = l; // HC1
    this.w = w; // HC2
  }
}

const rect1 = new RectArea(3,4)
const rect2 = new RectArea(5,6)
